# TinySoft 跨域数据补齐与提质计划（股票/基金/基金扩展/指数）- 2026-03-03

## 1. 范围与口径

- 计划日期：2026-03-03。
- 本计划仅覆盖非 Tick 粒度数据，继续遵循“分钟级行情 + 非行情高价值数据”路线。
- 目标不是重复建设 `tushare/akshare`，而是做两件事：
  - 补齐当前 `alphadb` 缺失且策略价值高的数据域。
  - 对 `tushare/akshare` 已有但口径/时点/细节不足的数据做提质增强。

### 2.2 `tushare` 与 `akshare`（股票/基金/指数）现状摘要

- `tushare.stock_*`、`fund_*`、`index_*` 主体表已较全，且多数更新到 2026-02-26/27。

- 明确缺口（当前库未见同类表）：
  - 北向链路细表（港股通/沪深股通日成交、活跃股、持股明细、卖空余量）。

  - 指数“市场交易日历”多市场口径（含北向/南向、港股、银行间）及指数财务聚合指标（754-763）。

## 5. 验收标准

- 覆盖率：新建表近 20 个交易日（日更表）或近 8 个披露周期（季更表）稳定更新。
- 完整性：主键重复率为 0，关键字段空值率低于设定阈值。
- 一致性：与 `tushare/akshare` 相似指标抽样偏差在阈值内；差异必须有口径解释。
- 可运维性：任务支持幂等重跑，质量检查结果可追踪落库。

## 6. 本轮结论

- 股票侧最优先是“北向 + 转融券”两条明确缺口链路。
- 基金侧最优先是“基金扩展（风格/风险/经理绩效）”，这是当前库明显空白。
- 指数侧最优先是“多市场交易日历 + 指数财务聚合指标”，可直接增强择时与风格轮动研究。
- 对已被 `tushare` 覆盖的表，不建议盲目重复入库，优先做字段差异驱动的提质落地。
