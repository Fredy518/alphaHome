# 系统模式（System Patterns）
创建时间：2025-08-02 12:52:02 CST

## 架构模式（Architectural Patterns）
- 模块化分层架构：采集（fetchers）、处理（factors/processors）、呈现（GUI）清晰分层与解耦
- 任务导向设计：通过显式任务定义与注册表（registry）提升可发现性与自动化能力
- 适配器模式：为外部数据源（Tushare、iFind）提供特定 API 封装与数据转换器
- 流水线模式：用于因子计算与研究工作流的组装与复用
- 集中式配置与日志：由 common 工具统一提供配置管理与日志能力
- 模式迁移（Schema Migration）：提供数据表结构演进的安全工具

## 横切关注点（Cross-Cutting Concerns）
- 稳健的日志与错误处理，覆盖长耗时的数据任务
- 幂等与可重试的外部 API 批量采集
- 领域模型与传输/数据帧表示之间的清晰分离
